1 Introduction

If desired, ELIAS can read all the information about a given theme from an XML document. This enables the composer to provide the required information in a convenient way, and saves the programmer the trouble of manually entering all the settings in the actual code.

Please note: This specification is not a tutorial. It tries to be as thorough and precise as possible, but assumes that the reader is already familiar with the concepts and terminology used in XML. It also does not explain in depth what the various settings do, merely what the allowed values are. For a detailed discussion on all the features and functionality found in ELIAS, please see the accompanying instruction manual.

Below is a comprehensive list of all the elements and their relations, as well as all the mandatory and optional attributes found in each element.

2 Elements

2.1 theme

The theme element is the top level element in every ELIAS project. All other elements are children of this element.

Mandatory attributes:

  • mode - Should be set to either objective or exploration.
  • tempo - Specifies the tempo of the theme in beats per minute (bpm).
  • time_signature - Gives the time signature as numerator/denominator, such as 4/4 or 3/4.
  • sample_rate - The number of samples per second used for the audio in the theme (44100 is the most common).
  • channels - The number of channels in the theme, usually set to 2 (stereo).
  • bars - Specifies how long the theme is in bars.

Optional attributes:

  • volume - The master volume for the entire theme, specified in millibel. Default is 0. -600 is -6 dB, 100 is +1 dB etc.

Mandatory child elements:

  • At least one track element (audio_loop_track or audio_stinger_track)

Optional child elements:

  • defaults
  • rendezvous
  • action_preset

2.2 defaults

This element specifies default settings that will be applied to any track that does not explicitly specify a new value for a given setting present in the defaults.

Permitted parent elements:

  • theme

Optional child elements:

  • fade_in
  • crossfade
  • fade_out
  • reverb
  • progression
  • Either agility or agility_points
  • pickup

2.3 rendezvous

Rendezvous settings come in effect when the engine needs to do a transition on all the tracks in the theme at the same time, such as when a key change occurs. It is preferable to specify separate agility and fade settings for these scenarios.

Permitted parent elements:

  • theme

Optional child elements:

  • fade
  • Either agility or agility_points

2.4 fade

This element is used to specify a fade length for rendezvous transitions.

Permitted parent elements:

  • rendezvous

Mandatory attributes:

  • value - The amount of time that the fade should take, in milliseconds. The minimum is 0.

2.5 reverb

The reverb element is used to specify the settings used for the reverb effect in the ELIAS engine.

Permitted parent elements:

  • defaults
  • audio_loop_track
  • audio_stinger_track

Optional attributes:

  • size - The size of the room, in percent. The minimum is 0 (smallest) and the maximum is 100 (largest).
  • damping - The damping of the room, in percent. The minimum is 0 (no damping) and the maximum is 100 (maximum damping).
  • mix - The mix between the reverberated and the dry signal, in percent. The minimum is 0 (completely dry) and the maximum is 100 (all reverb).

2.6 fade_in

This element specifies a range for a fade in operation, which occurs when an audio loop track goes from silence to having audio on it.

Permitted parent elements:

  • defaults
  • audio_loop_track

Optional attributes:

  • minimum - The shortest amount of time that the fade can take, in milliseconds. The minimum is 0.
  • maximum - The longest amount of time that the fade can take, in milliseconds. The minimum is 0.

2.7 crossfade

This element specifies a range for a crossfade operation, which occurs when an audio loop track goes from one source to another.

Permitted parent elements:

  • defaults
  • audio_loop_track

Optional attributes:

  • minimum - The shortest amount of time that the fade can take, in milliseconds. The minimum is 0.
  • maximum - The longest amount of time that the fade can take, in milliseconds. The minimum is 0.

2.8 fade_out

This element specifies a range for a fade out operation, which occurs when an audio loop track goes from having audio on it to silence.

Permitted parent elements:

  • defaults
  • audio_loop_track

Optional attributes:

  • minimum - The shortest amount of time that the fade can take, in milliseconds. The minimum is 0.
  • maximum - The longest amount of time that the fade can take, in milliseconds. The minimum is 0.

2.9 agility

This element specifies the so called "regular agility" setting.

Permitted parent elements:

  • defaults
  • rendezvous
  • audio_loop_track
  • audio_stinger_track

Mandatory attributes:

  • value - The agility period, in beats. The minimum is 0 (anytime) and the maximum is the number of bars in the theme times the time signature numerator.

Optional attributes:

  • strict - Specifies whether the engine should treat the agility setting as a suggestion or as an unconditional rule. The value should be either "yes" or "no". Default is "no". This attribute is ignored if the parent is a rendezvous element.

2.10 agility_points

This element specifies so called "agility beat points".

Permitted parent elements:

  • defaults
  • rendezvous
  • audio_loop_track
  • audio_stinger_track
  • stinger

Optional attributes:

  • strict - Specifies whether the engine should treat the agility beat point settings as a suggestion or as an unconditional rule. The value should be either "yes" or "no". Default is "no". This attribute is ignored if the parent is a rendezvous or a stinger element.

Mandatory child elements:

  • At least one (but usually several) beat elements

2.11 beat

This element specifies a single agility beat point.

Permitted parent elements:

  • agility_points

Mandatory attributes:

  • point - The agility beat point, in beats. The minimum is 1 and the maximum is the number of bars in the theme times the time signature numerator.

2.12 pickup

This element specifies a number of pickup beats for a stinger track.

Permitted parent elements:

  • defaults
  • audio_stinger_track

Mandatory attributes:

  • beats - The number of pickup beats. The minimum is 0 (no pickup beats) and the maximum is the number of bars in the theme times the time signature numerator, minus 1.

2.13 audio_loop_track

This element contains all the contents found on an audio loop track.

Permitted parent elements:

  • theme

Mandatory attributes:

  • id - A number which is unique among all the track elements in the theme, even if the types differ. The minimum is 1.

Optional attributes:

  • volume - The volume level for the track, specified in millibel. This volume level is relative to the master volume of the theme. Default is 0. -600 is -6 dB, 100 is +1 dB etc.

Mandatory child elements:

  • At least one key element

Optional child elements:

  • fade_in
  • crossfade
  • fade_out
  • reverb
  • Either agility or agility_points

2.14 key

This element contains the contents of a given key on a particular track.

Permitted parent elements:

  • audio_loop_track
  • audio_stinger_track

Mandatory attributes:

  • value - The number of semitones above C which this key represents. The minimum is 0 (C) and the maximum is defined by the elias_max_keys constant in elias.h.

Mandatory child elements:

The permitted child elements depend on the type of track in which this key element appears.

If the key element is a child of an audio_loop_track element, the following child elements are mandatory:

  • At least one trigger element

If the key element is a child of an audio_stinger_track element, the following child elements are mandatory:

  • At least one stinger element

2.15 trigger

This element represents a single trigger in a given key on an audio loop track in the theme.

Permitted parent elements:

  • key

Note: This element must only be present within a key if the parent element of that key is an audio_loop_track element.

Mandatory attributes:

  • value - A number which is unique in the given key on the given track. The minimum is 1.

Optional attributes:

  • length - The length of the given trigger audio source, in samples. The minimum is 0. If 0 is specified, the trigger is assumed to be exactly as long as the theme.
  • filename - The name of the file in which the audio data for this trigger can be found.
  • dummy - Specifies whether or not this is a silent dummy trigger with no audio attached to it. The value should be either "yes" or "no". Default is "no". If set to "yes", the filename and length attributes are ignored.

2.16 audio_stinger_track

This element contains all the contents found on an audio stinger track.

Permitted parent elements:

  • theme

Mandatory attributes:

  • id - A number which is unique among all the track elements in the theme, even if the types differ. The minimum is 1.

Optional attributes:

  • volume - The volume level for the track, specified in millibel. This volume level is relative to the master volume of the theme. Default is 0. -600 is -6 dB, 100 is +1 dB etc.

Mandatory child elements:

  • At least one key element

Optional child elements:

  • reverb
  • progression
  • Either agility or agility_points
  • pickup

2.17 stinger

This element represents a single stinger variation in a given key on an audio stinger track in the theme.

Permitted parent elements:

  • key

Note: This element must only be present within a key if the parent element of that key is an audio_stinger_track element.

Mandatory attributes:

  • value - A number which is unique in the given key on the given track. The minimum is 1.
  • length - The length of the given stinger audio source, in samples. The minimum is 1.

Optional attributes:

  • filename - The name of the file in which the audio data for this stinger can be found.

Optional child elements:

  • agility_points

2.18 progression

The progression setting controls how a stinger track will choose among the available variations when asked to play a new stinger.

Permitted parent elements:

  • defaults
  • audio_stinger_track

Mandatory attributes:

  • mode - A string specifying the desired progression mode ("random", "sequential" or "shuffle").

2.19 action_preset

This element specifies an action preset with one or more associated events.

Permitted parent elements:

  • theme

Mandatory attributes:

  • name - A unique name for the given action preset.

Optional child elements (at least one must be present):

  • set_level
  • silence
  • request_change
  • play_stinger
  • change_key

2.20 set_level

This element represents a level change event. This is only applicable in objective mode.

Permitted parent elements:

  • action_preset

Mandatory attributes:

  • minimum - The minimum value in the level range. Must be at least 1.
  • maximum - The maximum value in the level range (inclusive). This can be the same as minimum. Must be at least 1.

Optional attributes:

  • urgency - The urgency with which the transition should be performed in percent, between 0 and 100 (inclusive). Default is 100.
  • stinger - The id of a stinger track that should be associated with this transition. Default is 0 (no stinger).
  • rendezvous - Specifies whether this should be a rendezvous transition. The value should be either "yes" or "no". Default is "no".

2.21 silence

This element represents a silencing event. This is applicable both in objective and exploration mode.

Permitted parent elements:

  • action_preset

Optional attributes:

  • urgency - The urgency with which the transition should be performed in percent, between 0 and 100 (inclusive). Default is 100.
  • stinger - The id of a stinger track that should be associated with this transition. Default is 0 (no stinger).
  • rendezvous - Specifies whether this should be a rendezvous transition. The value should be either "yes" or "no". Default is "no".

2.22 request_change

This element represents a random change event. This is only applicable in exploration mode.

Permitted parent elements:

  • action_preset

Mandatory attributes:

  • track_count - The number of tracks that should be changed. Must be between 1 and the total number of loop tracks in the theme.

Optional attributes:

  • urgency - The urgency with which the transition should be performed in percent, between 0 and 100 (inclusive). Default is 100.
  • stinger - The id of a stinger track that should be associated with this transition. Default is 0 (no stinger).
  • rendezvous - Specifies whether this should be a rendezvous transition. The value should be either "yes" or "no". Default is "no".

2.23 play_stinger

This element represents a standalone stinger event. This is applicable both in objective and exploration mode.

Permitted parent elements:

  • action_preset

Mandatory attributes:

  • stinger - The id of the stinger track that should be triggered.

2.24 change_key

This element represents a key change event. This is applicable both in objective and exploration mode. Please note that the key change will not occur until the next transition (such as a level change) takes place.

Permitted parent elements:

  • action_preset

Mandatory attributes:

  • key - The new key that should be activated.

3 Conclusion

At this point you should be able to construct ELIAS XML documents to represent your themes with relative ease. It is equally trivial for the programmer to import your XML document into the ELIAS engine and get your theme up and running quickly and easily. XML is an invaluable communication bridge between the composer and the programmer, and we hope that it'll save everyone some time.
